# Contributing Guide (기여 가이드)

## 커밋 메시지 규칙 (한글 가능)
형식:
`타입: 요약`

타입 목록(권장):
- `feat`: 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 포맷/세미콜론/공백 등(로직 변경 없음)
- `refactor`: 리팩토링(기능 변화 없음)
- `perf`: 성능 개선
- `test`: 테스트 추가/수정
- `chore`: 빌드/설정/잡일

예시:
- `feat: 속도에 따른 FOV 변화 추가`
- `fix: 리사이즈 시 캔버스 비율 깨짐 수정`
- `docs: GitHub Pages 배포 문서 추가`
- `refactor: 렌더 루프 구조 정리`

권장 규칙:
- 한 커밋은 한 목적(작게)
- 요약은 명령문 형태(“추가”, “수정”, “정리” 등)

## PR 규칙
- PR 제목도 커밋 규칙을 따르는 것을 권장
  - 예: `feat: 스피드 라인 이펙트 구현`
- PR은 가능한 작게(리뷰 가능한 수준)
- PR에는 아래 내용을 포함:
  - 변경 내용 요약
  - 스크린샷/짧은 GIF(가능하면)
  - 테스트/빌드 결과 (`npm run build`)

## 머지 기준(권장)
- 빌드 성공
- 핵심 기능 동작 확인
- 콘솔 에러 없음(또는 원인/대응 문서화)
